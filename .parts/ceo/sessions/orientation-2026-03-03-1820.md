## Design

*Re-orientation — no new commits since cold boot at 6:20pm. Design picture unchanged. Cold boot analysis below remains current.*

**Current UX state:** Two surfaces exist: an Obsidian plugin (terminal tabs inside Obsidian) and a standalone Electron app (three-zone dispatch console). The Electron app is the focus surface. It uses a sidebar | focus-terminal | context-pane CSS Grid layout. Interaction is keyboard-first with Cmd-shortcuts for session management (T new, W close, Shift+[/] navigate, J next-attention, B toggle sidebar, \ toggle context, E rename, 1-9 jump-by-index). The terminal runs xterm.js with status detection (shell/active/tool/idle/waiting/error/exited) parsed from ANSI-stripped output. Orchestration activity (TeamCreate, SendMessage) is parsed and surfaced as team member chips and event summaries in sidebar cards and context pane.

**Interaction patterns observed:**
- **Session switching (Electron):** Cmd+Shift+[/] cycles sequentially with slide animation (translateX 40px, 200ms ease). Cmd+1-9 jumps by index. Fast and predictable for a keyboard user.
- **Attention queue:** Sessions needing attention (permission prompts, agent-finished-review) surfaced via yellow banner with count + Cmd+J to cycle. Attention clears on focus. This is the primary interrupt-driven navigation — the pattern that makes this a dispatch console rather than a tab bar.
- **Session rename:** Double-click in sidebar/focus header, or Cmd+E. Inline input, Enter commits, Escape cancels, blur commits. Works in both contexts.
- **Context pane:** Right column shows other sessions with status dot, status text, team event summary, member chips, and monospace preview of last output lines. Click card or member chip to switch. Peripheral awareness surface.
- **Terminal manager (Obsidian):** Flat list with status dots, names, unread badges, team metadata. Click to reveal leaf. Functional, simpler than the Electron sidebar.
- **Terminal switcher (Obsidian):** FuzzySuggestModal with status dot and name. Quick search-by-name. Minimal.

**Friction points:**
- **No "waiting" status in Obsidian plugin.** Electron detects permission/question prompts and flags "waiting" with attention escalation. Obsidian's `terminal-view.ts` has shell/active/tool/idle/exited only — no waiting state. Permission prompts on the Obsidian surface go unnoticed unless the user is looking at that tab. Largest attention-loss risk on the Obsidian surface.
- **Context pane preview is 10px monospace.** `context-card-preview` shows last output lines at 10px — borderline legible. The information is present but requires effort to read.
- **No keyboard shortcut discoverability.** Rich keyboard vocabulary with no help surface. Empty state shows "Cmd+T to create a session" but that's the only hint.
- **Member chip navigation silently fails.** If the target session hasn't been parsed yet, clicking a member chip does nothing. No toast, no feedback. The `showKeyHint()` function exists and could be used here.
- **ANSI stripping divergence.** Electron's `stripAnsi()` handles DEC private mode, character set selection, keypad mode, box drawing, block elements, private-use-area glyphs, and orphaned sequence tails. Obsidian's `stripAnsi()` handles only CSI, OSC, and charset. Status detection reliability differs between surfaces.
- **Attention signal disappears with sidebar collapse.** Toggling sidebar off hides the attention banner. No residual signal in focus header. Sessions needing attention while sidebar is collapsed produce no visible indicator.

**Design observations:**
- **Information hierarchy is sound.** Three zones separate inventory (sidebar), active work (focus), and awareness (context). Sidebar groups sessions by urgency (needs attention → working → other). Correct prioritization for dispatch.
- **Visual coherence between surfaces is intentionally divergent.** Electron uses bespoke Tokyo Night-adjacent CSS variables. Obsidian uses Obsidian theme variables. Both share naming vocabulary and status semantics. Acceptable for different host contexts. However, status dot color semantics differ: Electron maps `shell` to gray secondary and `idle` to gray dim; Obsidian maps `shell` to green and `idle` to blue. Same session state communicates differently depending on surface.
- **The app assumes continuous keyboard attention.** No notification mechanism beyond in-app attention banner (no sound, no system notification, no dock badge). For sessions running minutes at a time, the operator may look away. Pulsing yellow `waiting` dot is the strongest "come back" signal, and it requires the sidebar to be visible.
- **Mental model is clear.** Session-as-unit with name, status, optional team context. Attention queue adds a priority layer. A user watching this understands: "I have N sessions, some need me, most are working." Team metadata enriches without disrupting the core model.

**Recommendations:**
- **Port "waiting" status to Obsidian surface.** Add PERMISSION_PATTERN and QUESTION_PATTERN matching to `terminal-view.ts:detectStatus()`, a `waiting` TerminalStatus value, and corresponding CSS with pulsing animation. Closes the attention gap.
- **Surface attention state when sidebar is collapsed.** A count badge or dot on the collapsed sidebar edge, or in the focus header bar. The attention signal should survive sidebar collapse.
- **Add a shortcut overlay.** Cmd+/ or Cmd+? to show a transient overlay listing keyboard shortcuts. Low cost, eliminates discoverability gap.
- **Normalize ANSI stripping.** Extract Electron's comprehensive `stripAnsi()` to a shared module imported by both surfaces, so status detection behaves identically. (CTO flagged the same code duplication concern — this recommendation converges with theirs.)
- **Show feedback on failed member-chip navigation.** Use `showKeyHint("No session found for [name]")` instead of silent no-op.
- **Reconcile status dot color semantics.** `shell` and `idle` map to different colors across surfaces. Pick one mapping and apply to both so the same state looks the same regardless of surface.
