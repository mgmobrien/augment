## Scope expansion assessment: universal CC session visibility (6:40pm)

CEO asked for trajectory assessment of Matt's proposal to show ALL CC sessions on the machine (not just ones started inside Augment), including background automations and eventually remote VPS sessions.

**Web research conducted.** Scanned the current CC session monitoring landscape. Four tools doing some form of multi-session visibility:

1. **c9watch** (Tauri/Rust/SvelteKit) — OS-level process scanning via `sysinfo`, correlates with `~/.claude/projects/` session files. Groups by status (Working, Needs Permission, Idle, Done). Has permission-request surfacing and "jump to parent terminal" action. Closest competitor to this exact expansion. macOS menu bar app.

2. **claude-code-monitor** (Node/AppleScript) — uses CC hooks to log session data, shows in terminal UI + mobile web. Basic status indicators (running/waiting/done). No priority sorting. macOS only, 3 terminals supported.

3. **claude-dashboard** (Go) — tmux session lister. Requires tmux. No attention management. Viewer/switcher only.

4. **claude-code-karma** (local dashboard) — reads `~/.claude/` files for analytics. Observational only — sessions, costs, timelines. No dispatch, no control.

**Key finding:** c9watch is the only tool doing process-level discovery + attention-like grouping, but it's a *viewer* (menu bar popover, jump-to-terminal). It doesn't provide a dispatch surface — you still manage sessions in their original terminals. None of these tools combine discovery + attention prioritization + embedded terminal control in a single surface.

**Assessment:** This scope expansion strengthens the "attention dispatch" framing rather than complicating it. Detailed analysis sent to CEO.
